# 🚀 App Store / Google Play ストア出品 ロードマップ

現在の `React + Vite` で作成されたWebブラウザ向けの数独アプリを、iOS（App Store）およびAndroid（Google Play Store）のネイティブアプリとして公開するまでの全体ロードマップです。

現在のコードベースを活かし、最も少ない手間でネイティブ化できる **「Capacitor（キャパシター）」** を使用した方針を解説します。

---

## 🗺️ 全体フロー

1. **フェーズ1: アプリ化の準備（Capacitorの導入）**
2. **フェーズ2: モバイル向け最適化・テスト**
3. **フェーズ3: 開発者アカウントの登録**
4. **フェーズ4: アプリのビルドと公開準備**
5. **フェーズ5: ストア審査と公開**

---

## 🛠️ フェーズ1: アプリ化の準備（Capacitorの導入）

Webアプリを「iOS/Androidアプリ」の殻（WebViewと呼ばれるもの）で包み込むツールである `Capacitor` を導入します。

- [ ] **Capacitorのインストール**
  - プロジェクト内で `npm install @capacitor/core` および `@capacitor/cli` をインストールします。
- [ ] **Capacitorの初期化**
  - `npx cap init` コマンドを実行し、アプリ名（例: Sudoku Master）とバンドルID（例: `com.yourname.sudoku`）を設定します。
- [ ] **プラットフォームごとのパッケージ追加**
  - `npm install @capacitor/ios @capacitor/android`
  - `npx cap add ios` および `npx cap add android`
- [ ] **ビルドと同期**
  - Web画面をビルド（`npm run build`）し、`npx cap sync` でiOS/Androidのプロジェクトファイルに同期させます。

---

## 📱 フェーズ2: モバイル向け最適化・テスト

実際のスマートフォンやエミュレーターで正しく動くか、ブラウザ版との違いを調整します。

- [ ] **SafeArea（セーフエリア）の対応**
  - iPhoneのノッチ（画面の切り欠き）にUIが被らないように、一部のCSSに `env(safe-area-inset-top)` の調整を入れます。（現在のコードではすでに実装済みです！）
- [ ] **Haptic Feedback（バイブレーション）のネイティブ対応**
  - 現在の `navigator.vibrate` はiOSのSafariなど一部で動かないため、Capacitorの公式ハードウェアプラグイン（`@capacitor/haptics`）に置き換えます。
- [ ] **ローカルストレージの永続化確認**
  - 現在のセーブ・統計データは `localStorage` に保存されていますが、ネイティブアプリ化でもそのまま使用可能です。場合により `@capacitor/preferences` への置き換えを検討します。
- [ ] **デバイス・エミュレーターでの動作テスト**
  - Android Studio や Xcode を開き、仮想デバイス上でアプリを起動し、問題なく遊べるかテストします。

---

## 📝 フェーズ3: 開発者アカウントの登録

ストアにアプリを公開するためには、それぞれの公式開発者アカウントが必要です（審査があり、登録料がかかります）。

- [ ] **Apple Developer Program への登録 (iOS)**
  - **料金:** 年間 99 USD（毎年更新）
  - Apple IDを用意し、個人または法人として登録。二段階認証と身分証の確認が必要です。
- [ ] **Google Play Console への登録 (Android)**
  - **料金:** 25 USD（初回のみ買い切り）
  - Googleアカウントを使用して登録。本人確認（身分証のアップロード等）が行われます。

---

## 📦 フェーズ4: アプリのビルドと公開準備

実際にユーザーがストアで見るための画像や紹介文を用意し、アプリを提出用の形式にパッケージ化します。

- [ ] **アイコンとスプラッシュ画面（起動画面）の作成**
  - `cordova-res` や Capacitor 公式ツールを使って、数独アプリの正方形アイコンから一括で数十種類のサイズ別アイコンと起動画面を作成します。
- [ ] **ストア掲載情報の準備（スクリーンショット・説明文）**
  - アプリの魅力を伝えるスクリーンショット画像（iPhone用・iPad用・Android各種）を作成。
  - アプリのタイトル、キーワード、分かりやすい説明文（UIの美しさやSUPER HARDモードをアピール）を用意します。
- [ ] **プライバシーポリシーの作成**
  - アプリ用のプライバシーポリシーを作成し、ウェブ上のどこか（GitHub Pages等でOK）に公開してURLを用意します。
- [ ] **アプリの提出ビルド作成**
  - **iOS:** Xcodeから「Archive」を行い、App Store Connectへアップロードします。
  - **Android:** Android Studioから「App Bundle (.aabファイル)」を署名付きで生成し、Google Play Consoleへアップロードします。

---

## ⚖️ フェーズ5: ストア審査と公開

それぞれのストアへ提出し、Apple/Googleの審査官による審査を待ちます。

- [ ] **コンテンツのレーティング回答**
  - アプリ内の暴力表現や年齢制限のアンケートに回答します（数独なので全年齢対象になります）。
- [ ] **App Store 審査提出 (iOS)**
  - 審査ボタンを押し、完了を待ちます（通常24〜48時間程度）。リジェクト（却下）された場合は理由が送られてくるので、修正して再提出します。
- [ ] **Google Play 審査提出 (Android)**
  - Google Play Consoleで製品版のロールアウトを開始します（通常3〜7日程度かかります）。
  - ※ 2023年以降に新しく作った個人アカウントの場合、**20人のテスターに14日間継続してテストを行ってもらう**クローズドテストの条件が追加されていますので、テスター集めが必要になります。
- [ ] **🎉 リリース完了！**
  - 審査を通過すれば、世界中の人がApp Store / Google Playから数独アプリをダウンロードできるようになります！

---

## 💡 次のステップの提案

もしネイティブアプリ化を進める場合、まずは以下の第一歩から始めるのがお勧めです：
1. **Capacitorの導入（フェーズ1）**を行い、PC上で「アプリのガワ」を構築してみる。
2. Android Studio などを入れて、**自分のスマホにシミュレーターとしてインストール**してみる。

この作業であれば、開発者アカウントの料金を払う前の**無料の範囲**で全て試すことができます。
